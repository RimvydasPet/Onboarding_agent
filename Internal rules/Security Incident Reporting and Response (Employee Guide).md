# Security Incident Reporting and Response (Employee Guide)

## Purpose
Help employees recognize and quickly report security incidents to reduce impact.

## What counts as a security incident
- Suspected phishing or credential theft.
- Lost or stolen device.
- Malware alerts, unusual pop-ups, or unexpected encryption of files.
- Accidental data sharing (wrong recipient, public link, misconfigured folder).
- Unauthorized access or suspicious account activity.

## Immediate actions (do first)
1. If you clicked a suspicious link or entered credentials, report immediately.
2. Disconnect from the network if you suspect malware (turn off Wi‑Fi).
3. Do not delete emails, files, or logs that might be evidence.
4. Contact Security via: security-incident@example.invalid (or the hotline if provided).

## What to include in your report
- What happened and when.
- Screenshots (if safe) and email headers (for phishing).
- Systems involved (device name, application).
- Any data that may have been exposed.

## What not to do
- Do not “investigate” by forwarding suspicious emails to coworkers.
- Do not run unapproved cleanup tools.
- Do not post about the incident in public channels.

## Response overview (what Security will do)
- Triage severity and scope.
- Contain the issue (account reset, revoke tokens, quarantine device).
- Eradicate and recover (patch, restore, validate).
- Post-incident review for improvements.
