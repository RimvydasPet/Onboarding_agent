# Data Classification and Handling Standard

## Purpose
Define data classifications and handling rules to reduce legal, security, and operational risk.

## Scope
All information created, stored, processed, or transmitted on behalf of the company.

## Classifications
### Public
Information intended for public release (e.g., marketing website content).

### Internal
Information for employees/contractors that would cause limited harm if disclosed.

### Confidential
Information that could cause significant harm if disclosed (e.g., customer contracts, security details, employee data).

### Restricted
Highly sensitive information requiring strict controls (e.g., authentication secrets, security incident details, certain financial data).

## Handling rules
- Store Internal+ data only in approved company systems.
- Share Confidential/Restricted data externally only with a clear business need, approved method, and appropriate agreement (e.g., NDA) when required.
- Apply least-privilege access; remove access when no longer needed.

## Labeling
Documents should include classification in the header/footer or metadata where practical.

## Retention and deletion
- Keep data only as long as necessary for business and legal requirements.
- Use approved deletion methods; do not “self-retain” copies outside official systems.

## Incident considerations
Any suspected exposure of Confidential or Restricted data must be reported to Security immediately.

## Exceptions
Exceptions must be approved by Compliance Officer and Security Officer, documented with expiry and compensating controls.
