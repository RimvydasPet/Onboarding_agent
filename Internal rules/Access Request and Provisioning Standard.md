Access Request and Provisioning Standard

Purpose: Define a consistent process for requesting, approving, provisioning, reviewing, and removing access to company systems.

Scope: All business applications, infrastructure platforms, shared drives, and privileged access.

Roles and responsibilities
Requester: explains business need and desired access level.
Approver: validates need-to-know (typically manager or system owner).
IT: provisions access and records changes.
Security: approves privileged access and reviews audit logs.

Standard workflow
Submit request in the service portal with system name, role, and justification.
Manager approval for standard access.
System owner approval for application-specific roles.
Security approval for privileged/admin roles.
IT provisions access and confirms completion.
Access is reviewed periodically and removed when no longer required.

Least privilege rules
Start with the minimum role that supports duties.
Time-bound access for elevated permissions (e.g., 8 hours, 7 days).
Separate accounts for privileged actions where supported.

Emergency access (break glass)
Emergency access may be granted during incidents:
Requires on-call manager notification.
Must be logged and reviewed within 24 hours.
Automatically expires.

Offboarding
All access must be revoked on the user's last working day (or earlier if required). Managers must confirm transfer of ownership for shared resources.

Evidence and audit
Each access change must retain: request ticket, approvals, date/time, and operator identity.
